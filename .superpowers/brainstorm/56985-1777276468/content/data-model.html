Data Model

Two new Postgres tables for Track B — snapshots of every intelligence event, and the open positions they create

TABLE: intelligence_snapshots
column type notes
id UUID PK, gen_random_uuid()
ticker VARCHAR(10) null for macro events
sector VARCHAR(50) tech, energy, finance…
event_type intel_event_type enum ↓
source VARCHAR(30) capitol_trades, gdelt…
rule_score FLOAT always set, range −20/+20
llm_score FLOAT null if LLM not called
final_score FLOAT rule_score if no LLM
congressional_member VARCHAR(100) null for non-congress rows
trade_direction VARCHAR(4) buy / sell, null if N/A
trade_value_usd BIGINT in cents, null if N/A
summary TEXT human-readable headline
reasoning TEXT LLM chain-of-thought
raw_data JSONB original API payload
created_at TIMESTAMPTZ default now()
expires_at TIMESTAMPTZ now() + 90 days
ENUM: intel_event_type
congressional_trade
macro_news · gdelt_event
yahoo_news · social_sentiment
INDEXES
(ticker, created_at DESC)
(sector, created_at DESC)
(expires_at) — for cleanup job
(event_type, created_at DESC)
TABLE: intel_positions
column type notes
id UUID PK
ticker VARCHAR(10) not null
hold_type intel_hold_type enum ↓
status intel_pos_status enum ↓
size_usd DECIMAL(12,2) ≤ 40% of total capital
entry_price DECIMAL(12,4) fill price at open
entry_score FLOAT conviction at entry
snapshot_ids UUID[] which events triggered it
exit_price DECIMAL(12,4) null until closed
pnl_usd DECIMAL(12,2) null until closed
opened_at TIMESTAMPTZ default now()
closed_at TIMESTAMPTZ null until closed
intel_hold_type
swing
weekly
monthly
intel_pos_status
open
reduced
closed
INDEXES
(ticker, status) — active lookup
(status, hold_type) — dashboard filter
(opened_at DESC) — history view
CAPITAL CONSTRAINT
Before opening: SELECT SUM(size_usd)
FROM intel_positions WHERE status='open'
→ must stay ≤ 40% of total capital
Checked inside a DB transaction
