LLM Reasoning Service

How the Gemini → Groq → OpenRouter fallback chain works — and what happens when every provider is down

CALL FLOW
# rule scorer always runs first (free, instant)
rule_score = rule_scorer.score(events)

# LLM only called on high-conviction events
if abs(rule_score) < THRESHOLD (8.0):
  return rule_score  # skip LLM

# fallback chain — try each provider
for provider in [gemini, groq, openrouter]:
  try:
    result = await provider.analyze(ctx)
    validate_output(result)  # schema + range
    return merge(rule_score, result)
  except (ProviderError, ValidationError):
    continue  # try next

# all three failed
raise AIUnavailableError()
# executor catches → NO new positions opened
THRESHOLD LOGIC
rule_score < 8.0 → rule-only (most events)
rule_score ≥ 8.0 → call LLM for deep reasoning
~5–15 LLM calls/day, well within 1,500 free limit
SCORE MERGE
final = 0.4 × rule_score + 0.6 × llm_score
LLM weighted higher — it has full context
Both scores stored separately for audit
INPUT / OUTPUT CONTRACT
IntelContext
  # what gets sent to every provider
  ticker: str | None
  sector: str | None
  rule_score: float
  events: list[EventSummary]
    # max 10 recent events, text-only

LLMAnalysis
  # what every provider must return
  score: float  # −10.0 to +10.0
  reasoning: str  # non-empty
  provider: str  # which one answered
PROVIDERS
1. Gemini 2.0 Flash — primary
Free: 1,500 req/day · 1M tokens/min
Library: google-generativeai
Prompt → JSON mode → parse → validate
2. Groq / Llama 3.1 70B — fast fallback
Free: 14,400 req/day · very fast inference
Library: groq (OpenAI-compatible SDK)
Same JSON prompt schema as Gemini
3. OpenRouter — last resort
Free models: Mistral 7B, Phi-3
Library: openai (OpenRouter is API-compatible)
Same JSON prompt schema
All fail → AIUnavailableError
Caught by intel_portfolio_executor
Sets intel:ai_status = "down" in Redis
No new Track B positions opened
Dashboard shows red AI status badge
OUTPUT VALIDATION — what counts as a provider failure
HTTP error / timeout
network or rate limit
JSON parse failure
malformed response
score outside [−10, +10]
hallucination guard
Any of these → log warning → move to next provider. Never suppress or default-to-zero.
